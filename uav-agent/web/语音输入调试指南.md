# 语音输入调试指南

## 🔍 问题现象
点击录音 → 录音结束 → 点击解析指令 → 提示"请输入飞控指令"

## 📋 排查步骤

### 步骤 1：打开浏览器开发者工具

1. 在浏览器中按 `F12` 打开开发者工具
2. 切换到 **Console（控制台）** 标签
3. 清除控制台内容

### 步骤 2：测试语音识别

1. 点击麦克风按钮开始录音
2. 对着麦克风清楚地说："起飞到软件园三期F06"
3. 再次点击麦克风按钮停止录音
4. 观察控制台输出

### 步骤 3：查看日志输出

正常的日志输出应该类似这样：

```
[语音识别] 开始录音
[语音识别] final: 起飞到软件园三期F06 interim: 
[语音识别] 停止录音，当前输入框内容: 起飞到软件园三期F06
[语音识别] 录音结束
```

### 步骤 4：根据日志判断问题

#### 情况 A：没有看到任何日志
**原因：** 语音识别根本没有启动
**解决方案：**
- 检查浏览器是否支持 Web Speech API
- 在控制台手动输入：`window.SpeechRecognition || window.webkitSpeechRecognition`
- 如果返回 `undefined`，说明浏览器不支持，请换用 Chrome 浏览器

#### 情况 B：看到"开始录音"，但没有"final"或"interim"日志
**原因：** 麦克风没有接收到声音
**解决方案：**
- 检查麦克风权限是否允许
- 检查麦克风是否正常工作
- 尝试在控制台输入：`navigator.mediaDevices.getUserMedia({audio: true})` 测试麦克风

#### 情况 C：看到"interim"日志，但内容为空或乱码
**原因：** 语音识别引擎问题
**解决方案：**
- 确保网络连接正常（语音识别需要联网）
- 说话更清晰、更大声
- 尝试使用 Chrome 浏览器

#### 情况 D：看到"final"日志，有正确内容，但输入框仍为空
**原因：** Vue 数据绑定问题
**解决方案：**
- 刷新页面重试
- 检查是否有 JavaScript 错误

#### 情况 E：看到"未识别到语音内容"提示
**原因：** 录音时间太短或没有说话
**解决方案：**
- 录音后至少说一句话
- 确保说话时麦克风在工作

## 🛠️ 手动测试

如果语音识别有问题，可以在控制台手动测试：

### 测试 1：检查浏览器支持
```javascript
const SR = window.SpeechRecognition || window.webkitSpeechRecognition
console.log('浏览器支持语音识别:', !!SR)
```

### 测试 2：手动启动语音识别
```javascript
const recognition = new (window.SpeechRecognition || window.webkitSpeechRecognition)()
recognition.lang = 'zh-CN'
recognition.onresult = (e) => console.log('识别结果:', e.results[0][0].transcript)
recognition.start()
```

### 测试 3：检查输入框绑定
```javascript
// 在页面输入框输入一些文字，然后在控制台执行：
document.querySelector('textarea').value
```

## 📱 手机端调试

### Android Chrome
1. 手机连接电脑
2. 在电脑 Chrome 访问：`chrome://inspect`
3. 找到你的手机设备，点击 "inspect"
4. 可以查看手机浏览器的控制台日志

### iPhone Safari
1. 在 iPhone 上：设置 → Safari → 高级 → 启用"网页检查器"
2. 在 Mac Safari：开发菜单 → 选择你的 iPhone
3. 查看控制台日志

## 🐛 常见问题

### Q1: 录音按钮点击没反应
**检查：**
- 控制台是否有错误
- `speechSupported` 是否为 `true`
- 浏览器是否支持语音识别

### Q2: 录音后输入框没有文字
**检查：**
- 控制台是否有 `[语音识别]` 开头的日志
- 是否有 `final` 或 `interim` 结果
- 网络是否正常

### Q3: 识别的文字不正确
**解决：**
- 说话更清晰、语速放慢
- 在安静环境测试
- 识别后可以手动修改输入框内容

### Q4: 提示"请允许麦克风访问权限"
**解决：**
- 点击浏览器地址栏的权限图标
- 允许麦克风访问
- 刷新页面重试

## 💡 临时解决方案

如果语音识别暂时无法使用，可以：

1. **手动输入** - 直接在输入框打字
2. **语音转文字工具** - 使用手机输入法的语音输入，然后复制粘贴
3. **换个浏览器** - 推荐使用 Chrome 最新版
