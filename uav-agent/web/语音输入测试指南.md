# 语音输入功能 - 真实设备测试指南

## 📱 测试准备

### 1. 访问地址

在你的手机或其他设备上，打开浏览器访问以下地址（选择与你设备在同一网络的IP）：

```
http://172.19.224.1:8081
或
http://192.168.0.102:8081
```

### 2. 浏览器要求

- ✅ **Chrome for Android** - 完全支持
- ✅ **Safari for iOS** - iOS 14.5+ 支持
- ⚠️ 其他浏览器 - 可能不支持语音识别

### 3. 重要：HTTPS 要求

⚠️ **现代浏览器要求语音识别必须在 HTTPS 环境下使用**

如果你在手机上访问 `http://` 地址，浏览器可能会阻止麦克风访问。

**解决方案（任选其一）：**

#### 方案 A：使用 Chrome 的 localhost 隧道（推荐）
在你的电脑上运行：
```bash
npx localtunnel --port 8081
```
这会生成一个 HTTPS 的公网 URL，可以在手机上访问。

#### 方案 B：启用开发服务器的 HTTPS
修改 `vue.config.js`：
```javascript
devServer: {
  https: true,
  // ... 其他配置
}
```
然后访问 `https://192.168.0.102:8081`（需要接受自签名证书警告）

#### 方案 C：使用 Chrome 调试模式（仅 Android）
在手机上打开 Chrome，访问：
```
chrome://flags/#unsafely-treat-insecure-origin-as-secure
```
启用该选项并添加：
```
http://192.168.0.102:8081
```
重启 Chrome 后即可测试。

## 🧪 测试步骤

### 步骤 1：基础功能测试

1. 打开页面，检查是否显示 🎤 麦克风图标
2. 点击麦克风按钮
3. 浏览器会请求麦克风权限 → 点击"允许"
4. 对着设备说话："起飞到软件园三期F06"
5. 观察输入框是否自动填充文字
6. 再次点击麦克风按钮停止录音

测试以下常用指令：

| 测试指令 | 预期识别结果 |
|---------|------------|
| 起飞到软件园三期F06，高度120米 | ✅ 准确识别地址和高度 |
| 飞到F09栋，拍照后返航 | ✅ 识别多个连续指令 |
| 云台对准目标，开始录像 | ✅ 识别设备控制指令 |
| 打开探照灯，亮度调到80 | ✅ 识别参数设置 |
| 前进100米，然后返航 | ✅ 识别距离和动作 |

### 步骤 3：连续语音测试

1. 点击麦克风开始录音
2. 连续说出长指令：
   > "起飞到软件园三期F06，高度120米，飞到F09栋，拍照，然后返航"
3. 观察是否完整识别所有内容
4. 检查是否自动追加到输入框

### 步骤 4：错误处理测试

| 测试场景 | 预期行为 |
|---------|---------|
| 拒绝麦克风权限 | 显示"请允许麦克风访问权限" |
| 不说话 | 显示"未检测到语音" |
| 快速点击开始/停止 | 不崩溃，正常切换状态 |
| 网络断开 | 优雅降级，不影响手动输入 |

## 🐛 常见问题排查

### 问题 1：看不到麦克风图标
**原因：** 浏览器不支持 Web Speech API
**解决：** 使用 Chrome 或 Safari 浏览器

### 问题 2：点击麦克风无反应
**原因：** 麦克风权限被拒绝
**解决：** 
- 检查浏览器地址栏的权限图标
- 点击并允许麦克风访问
- 刷新页面重试

### 问题 3：识别结果为空
**原因：** 
- 说话声音太小
- 环境噪音太大
- 网络延迟

**解决：**
- 靠近麦克风说话
- 在安静环境测试
- 等待几秒再试

### 问题 4：识别文字不正确
**原因：** 语音识别引擎对专业术语不熟悉
**解决：**
- 说话更清晰、更慢
- 使用常用词汇替代（如"飞行"代替"飞控"）
- 识别后可手动编辑修正
