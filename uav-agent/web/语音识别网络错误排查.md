# 语音识别 Network 错误排查指南

## 🔍 错误信息
```
语音识别失败: network
```

## 📋 问题原因

这个错误表示语音识别服务无法连接到网络。Web Speech API 需要将音频数据发送到云端进行识别，因此必须联网。

---

## ✅ 解决方案（按优先级排序）

### 方案 1：检查网络连接

#### 步骤：
1. **确认设备已连接互联网**
   - 电脑/手机是否连接 WiFi 或移动数据？
   - 尝试打开网页：`https://www.baidu.com`

2. **测试 DNS 解析**
   ```bash
   # Windows
   ping www.baidu.com
   
   # 如果能 ping 通 IP 但 ping 不通域名，是 DNS 问题
   ```

3. **检查防火墙/代理**
   - 是否使用了公司网络？可能被防火墙阻止
   - 是否设置了代理服务器？尝试关闭代理

---

### 方案 2：使用 HTTPS（重要！）

#### ⚠️ 关键问题
**现代浏览器（尤其是 Chrome）要求语音识别必须在 HTTPS 环境下使用！**

如果你在手机上访问 `http://192.168.x.x:8081`，浏览器可能会：
- 阻止麦克风访问
- 阻止语音识别服务连接
- 报 `network` 错误

#### 解决方法：

**方法 A：使用本地隧道（推荐，最简单）**

1. 停止当前开发服务器（Ctrl+C）
2. 安装并运行 localtunnel：
   ```bash
   npx localtunnel --port 8081
   ```
3. 会生成一个 HTTPS 网址，如：`https://xxx-xxx-xxx.loca.lt`
4. 在手机浏览器访问这个 HTTPS 链接

**方法 B：启用开发服务器 HTTPS**

修改 `vue.config.js`：
```javascript
devServer: {
  https: true,  // 添加这行
  host: '0.0.0.0',
  port: 8081,
  // ... 其他配置
}
```

然后访问：`https://192.168.0.102:8081`
（浏览器会提示证书不安全，点击"继续访问"即可）

**方法 C：Android Chrome 特殊设置**

1. 在 Chrome 地址栏输入：`chrome://flags/#unsafely-treat-insecure-origin-as-secure`
2. 启用该选项
3. 添加：`http://192.168.0.102:8081`
4. 重启 Chrome
5. 再次访问测试

---

### 方案 3：更换浏览器

不同浏览器对语音识别的支持不同：

| 浏览器 | 语音识别支持 | 备注 |
|--------|------------|------|
| Chrome (PC) | ✅ 完全支持 | 推荐 |
| Chrome (Android) | ✅ 完全支持 | 推荐 |
| Edge | ✅ 支持 | 基于 Chromium |
| Safari (iOS) | ✅ 支持 | iOS 14.5+ |
| Firefox | ⚠️ 有限支持 | 可能有问题 |
| 微信内置浏览器 | ❌ 不支持 | 请使用系统浏览器 |
| QQ 内置浏览器 | ❌ 不支持 | 请使用系统浏览器 |

**建议：使用最新版 Chrome 浏览器**

---

### 方案 4：检查设备时间和时区

语音识别服务会验证 SSL 证书，如果设备时间不正确，证书验证会失败。

#### 检查步骤：
1. 打开设备设置
2. 检查日期和时间是否正确
3. 建议开启"自动设置时间"
4. 检查时区是否正确（中国应为 UTC+8）

---

### 方案 5：清除浏览器缓存

有时浏览器缓存会导致问题：

1. 按 `Ctrl + Shift + Delete`
2. 清除"缓存的图片和文件"
3. 刷新页面重试

## 🧪 诊断测试

### 测试 1：检查网络连接
在浏览器控制台执行：
```javascript
fetch('https://www.baidu.com')
  .then(() => console.log('✅ 网络正常'))
  .catch(err => console.log('❌ 网络错误:', err))
```

### 测试 2：检查语音识别服务
```javascript
const SR = window.SpeechRecognition || window.webkitSpeechRecognition
if (!SR) {
  console.log('❌ 浏览器不支持语音识别')
} else {
  console.log('✅ 浏览器支持语音识别')
  
  const recognition = new SR()
  recognition.lang = 'zh-CN'
  recognition.onresult = (e) => console.log('识别成功:', e.results[0][0].transcript)
  recognition.onerror = (e) => console.error('识别失败:', e.error)
  recognition.start()
  
  console.log('请说话...')
}
```

### 测试 3：检查当前协议
```javascript
console.log('当前协议:', window.location.protocol)
console.log('是否 HTTPS:', window.location.protocol === 'https:')
console.log('是否 localhost:', window.location.hostname === 'localhost' || window.location.hostname === '127.0.0.1')
```

**预期结果：**
- `当前协议: https:` 或 `当前协议: http:` 但 `是否 localhost: true`
- 如果是 `http:` 且不是 localhost，语音识别可能会被阻止

## 📱 手机端特殊处理

### Android
1. 确保使用 Chrome 浏览器
2. 确保已连接 WiFi 或移动数据
3. 如果访问 HTTP 地址，按照方案 2 的方法 C 设置
4. 清除 Chrome 应用缓存：设置 → 应用 → Chrome → 存储 → 清除缓存

### iPhone
1. 确保使用 Safari 浏览器
2. iOS 版本需要 14.5 或更高
3. 检查：设置 → Safari → 高级 → 确保"网页检查器"已启用
4. 尝试：设置 → Safari → 清除历史记录与网站数据

如果你想在本地快速测试，最简单的方法：

```bash
# 1. 确保在电脑上（不是手机）
# 2. 使用 localhost 访问
http://localhost:8081

# localhost 被视为安全环境，即使是 HTTP 也能使用语音识别
```

## 💡 临时替代方案

如果网络问题暂时无法解决，可以：

1. **使用手动输入** - 直接在输入框打字
2. **使用输入法语音输入** - 
   - Android: Gboard 输入法的麦克风图标
   - iOS: 系统键盘的麦克风图标
   - Windows: Win + H 快捷键
3. **复制粘贴** - 用其他语音转文字工具识别后粘贴
